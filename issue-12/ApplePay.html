Apple Pay

payment system from Apple
ရေးသားသူ : saturngod

Apple က payment system အတွက် Apple Pay ဆိုပြီး iPhone 6 , iPhone 6 plus မှာ ပါဝင်လာပါတယ်။ Apple Pay အကောင်း မပြောခင်မှာ အခု Payment တွေ ဘယ်လို လုပ်နေလဲ ဆိုတာကို တချက် ကြည့် ရအောင် ။

How it works now

Payment လုပ်တဲ့ အခါမှာ card data တွေကို encrypted လုပ်ပြီးတော့ Merchant Bank ( Acquirer ) ဆီကို ပို့လိုက်ပါတယ်။ Acquirer ဆီ ရောက်တော့ card network ကနေ တဆင့် customer bank ( issuer ) ကို ထပ်ပြီး request ထပ်လုပ်တယ်။ issuer က ရောက်လာတဲ့ card ကို accept သို့မဟုတ် denise လုပ်ပြီးတော့ status code ကို လာတဲ့ လမ်းအတိုင်း ပြန်ပြီးတော့ ပို့ပေးပါတယ်။ အများအားဖြင့် လုံခြုံစိတ်ချရမှု ရှိပါတယ်။

How Apple Pay is different

Apple Pay ကနေ tokenization ကို အသုံးပြုထားပါတယ် ။ tokenization ဟာ payemnt လုပ်တဲ့ အခါမှာ တကယ့်လက်ရှိ card number ကို speical number တွေ နဲ့ ပြောင်းထားပါတယ်။ Tokenization ဟာ ပိုပြီး လုံခြုံစိတ်ချရမှု ရှိပါတယ်။ သမာရိုးကျ ဆိုရင်တော့ Tokenization ဟာ payment gateway (PGW) မှာ card number တွေ expiry date တွေကို သိမ်းထားပြီးတော့ Tokenization ကို PGW database နဲ့ ချိတ်ဆက်ပြီးတော့ ထုတ်ပေးပါတယ်။

Apple Pay ဟာ Visa , American Express စတဲ့ credit card networks တွေနဲ့ ချိတ်ဆက်ထားပြီးတော့ Tokenization ကို card networks တွေက သိမ်းထားပါတယ်။ တကယ်လို့ ဖုန်းပျောက်သွားခဲ့ရင် Find My iPhone ကနေ lost mode ကို on လိုက်တာနဲ့ Apple က ချက်ခြင်း Token ကို ရှင်းလိုက်မှာ ဖြစ်တဲ့ အတွက် ဖုန်းပျောက်ရင် ကဒ်တွေ ပါသွားမှာ မပူရပါဘူး။ Find My iPhone ကိုတော့ on ထားပေးဖို့လိုပါလိမ့်မယ်။

Network Token ဟာ ပုံမှန် PAN (card number) လိုမျိုးပါပဲ။ 16 digits ရှိပြီးတော့ လက်ရှိ payment procession infrastructure တွေ နဲ့ အဆင်ပြေပါတယ်။ နောက်ပြီးတော့ Payment Network (Visa,Master) ကသာ သိမ်းထားတဲ့ အတွက် ပိုပြီး လုံခြုံစိတ်ချရမှုရှိပါတယ်။ Token ကို iPhone မှာ Secure Element မှာ သီးသန့် သိမ်းထားပါတယ်။ တခြား storage တွေထက် ပိုပြီး လုံခြုံစိတ်ချရမှု ရှိတယ်။

Apple Pay For Developer

Apple ဟာ payment အတွက် EMV နဲ့ 3-D Secure ကို အသုံးပြုထားပါတယ်။ ပုံမှန် Developer အနေနဲ့ Apple Pay support လုပ်တဲ့ Payment Gateway ကို အသုံးပြုနိုင်ပါတယ်။ Payment Gateway တွေ အနေနဲ့လည်း Apple Pay ကို အသုံးပြုပြီးတော့ ကိုယ်ပိုင် SDK ထဲမှာ ထည့်သွင်းနိုင်ပါတယ်။ Apple Pay ကို အသုံးပြုတဲ့အတွက် transaction fee တွေ ပေးဖို့ မလိုပါဘူး။ လက်ရှိ Payment Gateway တွေ အနေနဲ့လည်း Apple Pay ကို ပေါင်းထည့်တဲ့ အတွက် ဘာ fee မှ ပေးဖို့ မလိုပါဘူး။ Payment Gateway တွေ အနေနဲ့ Apple Pay Integration လုပ်ဖို့ အတွက် Apple Pay မှာ သွားပြီး ဖတ်ရှုနိုင်ပါတယ်။

Apple Pay ကို Test လုပ်ဖို့ အတွက် US credit card ရှိဖို့ လိုအပ်ပါတယ်။ ကိုယ့်ရဲ့ ဖုန်းကို Region မှာ United State ပြောင်းထားလိုက်ရင် ရပါပြီ။ Passbook app ကနေ တဆင့် Card ကို ထည့်သွင်းလို့ ရပါတယ်။ ပြီးတာနဲ့ Development အတွက် စပြီး စမ်းလို့ ရပါပြီ။

ရေးသားဖို့အတွက် Apple Pay မှာ မှာ သွားကြည့်နိုင်သလို Strip မှာ လည်း သွားပြီး ဖတ်ရှုနိုင်ပါတယ်။ Strip မှာတော့ Strip နဲ့ Integration လုပ်ဖို့အတွက် ရေးသားထားပေမယ့် Flow ကိုတော့ လေ့လာနိုင်ပါတယ်။
