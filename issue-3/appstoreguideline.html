App Store Review Guidelines(Part 1)

App Store ပေါ်မတင်ခင် သိသင့်သည့် စည်းမျဉ်းများ
ရေးသားသူ : ဇော်မင်းစိုး

ဆောင်းပါးဟာ အတန်ငယ် ရှည်လျားတဲ့အတွက်ကြောင့် အပိုင်းလေးတွေ ခွဲရေးဖို့ စိတ်ကူးထားပါတယ်။ ဒါကြောင့် ယခုလအတွက်တော့ အပိုင်း (၁) ဖြစ်တဲ့ “App Store ပေါ်မှ စည်းမျဉ်းများ အပိုင်း (၁)” ကိုကျွန်တော်တို့ စတင်လေ့လာကြပါမယ်။

Introduction

မင်္ဂလာပါ။

iOS အတွက် မိတ်ဆွေရဲ့ အချိန်နဲ့ စွမ်းရည်များကို အသုံးပြုပြီး Mobile App ဖန်တီးတဲ့အတွက် Apple မှကျေးဇူးတင်ပါတယ်။

ကျွန်တော်တို့ရဲ့ Apple Store ပေါ်မှာ မြောက်များစွာသော App တွေ. သီချင်းတွေ.. စာအုပ်တွေရှိပါတယ်။ထို့ကြောင့် ကျွန်တော်တို့အနေနဲ့ Store ပေါ်မှာ မြောက်များစွာသော Categories တွေကို စည်းစနစ်ကျစွာ စီမံထားရှိဖို့အတွက် စည်းကမ်းစနစ်တွေကို ချမှတ်ခဲ့ပါတယ်။ အဲ့ဒီအချက်အလက်တွေကို ကျေးဇူးပြုပြီး လိုက်နာပြီးမှသာ Store ပေါ်သို့တင်ပေးမှာဖြစ်ပါတယ်။ မလိုက်နာတဲ့ App တွေကိုတော့ ကျွန်တော်တို့ Apple Review Team အနေနဲ့ ပယ်သွားမှာဖြစ်ပါတယ်။

App Store ပေါ်မှာ မြောက်များစွာသော App တွေဟာ ရှိနေပါတယ်။ အရေအတွက် အားဖြင့်ဆိုရင် 350,000 apps ထက်ပိုသော အရေအတွက်အားဖြင့် App Store ပေါ်မှာ App တွေရှိပါတယ်။ဒါကြောင့် App Store ပေါ်မှာ အသုံးမဝင်တဲ့ App တွေရှိနေမှာကို ကျွန်တော်တို့အနေနဲ့ မလိုလားပါဘူး။

မိတ်ဆွေရဲ့ App ဟာ User တွေအတွက် အသုံးဝင်တဲ့ လုပ်ဆောင်ချက်တွေမပါပဲနဲ့ သာမန် ပြီးပြီးရော App ဖြစ်နေရင်တော့ ကျွန်တော်တို့အနေနဲ့ လက်ခံသွားမှာ မဟုတ်ပါဘူး။

ဒါကြောင့် အသုံးမဝင်တဲ့ App တွေ ၊ ဆင်တူရိုးမှား မြောက်များစွာ ရှိသော App တွေ… ဒါ့အပြင် ကျွန်တော်တို့ရဲ့ စည်းကမ်းတွေကို မလိုက်နာတဲ့ App တွေရှိခဲ့ရင်တော့ ကျွန်တော်တို့အနေနဲ့ Reject ပြုလုပ်သွားမှာဖြစ်ပါတယ်။

အကယ်၍သင့်ရဲ့ App ဟာ Reject ဖြစ်သွားပြီဆိုရင်တော့ ကျွန်တော်တို့အနေနဲ့ Review Board မှသင့်ရဲ့ Reject Appeal တင်ပြမှုကို လက်ခံစဉ်းစားပေးသွားမှာ ဖြစ်ပါတယ်။ စဉ်းစားပေးပြီး လက်သင့်ခံနိုင်တဲ့ အခြေအနေမှာ ပြန်လည် လက်ခံပေးသွားမှာဖြစ်ပါတယ်။ ထိုသို့မှမဟုတ်ပဲ ကျွန်တော်တို့ရဲ့ Apple ကို ထိခိုက်အောင် ၊ သိက္ခာကျအောင် ၊ တစ်စုံတစ်ရာ ပြောလာခဲ့ရင်တော့ သင့်ရဲ့ App အတွက် ဘယ်တော့မှ ပြန်လည် အဖတ်ဆယ်လို့ရတော့မှာ မဟုတ်ပါဘူး။

အကယ်၍ သင့်ရဲ့ App ထဲမှာ Cheat System တွေ (eg.User ဆီမှ password ခိုးယူခြင်း ၊ အခြား Developer ထံမှ အချက်အလက်ကူးယူခြင်း) ပါလာခဲ့ရင်တော့ ကျွန်တော်တို့အနေနဲ့ App Store ပေါ်ကို တင်ဖို့ရာ အတွက် ငြင်းပယ်သွားမှာ ဖြစ်ပါတယ်။

ဒါကြောင့် ကျေးဇူးပြုပြီး ကျွန်တော်တို့ ရဲ့ စည်းမျဉ်း စည်းကမ်းတွေကို လိုက်နာပြီးမှာ App Store ပေါ်ကိုတင်ဖို့ရာ အောင်မြင်မှာဖြစ်တဲ့ အတွက်ကြောင့် အောက်ဖော်ပြပါ အချက်အလက်တွေကို တိတိကျကျ လိုက်နာဆောင်ရွက်ဖို့ လိုမှာဖြစ်ပါတယ်။

1. Term and Conditions

  • 1.1 Apple App Store အတွက် Developer သည် Apple ၏ PLF (Program License Agreement) , HIG (Human Interface Guideline) နှင့် အခြားသော အချက်အလက်များကို Apple နဲ့စာချုပ်ချုပ်ပြီးကတည်းက သဘောတူပြီးဖြစ်ပါသည်။အောက်ဖော်ပြပါ အချက်အလက်တွေသည် သင်၏ App ကို Apple App Store ပေါ်သို့တင်ဆောင်ရာတင် အောင်မြင်မှု ရှိစေရန် လိုက်နာဆောင်ရွက်ရမည့် စည်းမျဉ်း စည်းကမ်းများသာဖြစ်ပါသည်။

2. Functionality

  • 2.1 သင်၏ App သည် အသုံးပြုစဉ် Crash ဖြစ်ပါက ငြင်းပယ်ခြင်း ခံရမည်။

  • 2.2 သင်၏ App မှ Bug များကို ဖော်ပြထားပါက ငြင်းပယ်ခြင်း ခံရမည်။

  • 2.3 သင်၏ App သည် သင့်၏ Development ကြော်ငြာမှ အပ အခြားသော ကြော်ငြာများပါလျှင် ငြင်းပယ်ခြင်း ခံရမည်။

  • 2.4 သင်၏ App သည် Review တင်စဉ်အချိန်တွင် ဖော်ပြထားသော Description နှင့် မကိုက်ညီသော လျှို့ဝှက်ထားသည့် လုပ်ဆောင်ချက်များ ပါရှိပါက ငြင်းပယ်ခြင်းခံရမည်။

  • 2.5 သင်၏ App သည် Apple မှသတ်မှတ်ထားသော Public API မဟုတ်သော API များကို အသုံးပြုထားပါက ငြင်းပယ်ခြင်း ခံရမည်။

  • 2.6 သင်၏ App သည် တားမြစ်ထားသော အခြား ပြင်ပမှ Data များကို Read လုပ်ခြင်း (သို့) Write လုပ်ခြင်း ပါဝင်ပါက ငြင်းပယ်ခြင်း ခံရမည်။

  • 2.7 သင်၏ App သည် ပြင်ပမှ Code များကို ဒေါင်းလုပ်လုပ်ခြင်း. (သို့) ထိုကဲ့သို့ လုပ်ဆောင်ချက်များပါဝင်ပါက ငြင်းပယ်ခြင်း ခံရမည်။

  • 2.8 သင်၏ App သည် သင်၏ Code များမှအပ အခြားသော ပြင်ပမှ Code များကို Install လုပ်ခြင်း (သို့) ထည့်သွင်းပါက ငြင်းပယ်ခြင်းခံရမည်။

  • 2.9 သင်၏ App သည် Beta Version (သို့) Demo Version (သို့) Trial Version (သို့) Testing Version များဖြစ်ပါက ငြင်းပယ်ခြင်းခံရမည်။

  • 2.10 သင်၏ App သည် iPhone only အတွက် ရေးသားထားသော်လည်း iPad တွင် မည်သည့် ပြင်ဆင်မှုမှမပါပဲ (2x iPhone 3GS Resolution) ဖြင့် အသုံးပြုနိုင်အောင် လုပ်ဆောင်ထားရမည်ဖြစ်သည်။

  • 2.11 သင်၏ App သည် App Store ပေါ်တွင် ရှိသော App တစ်ခုခုနှင့် Duplicate ဖြစ်ပါက ငြင်းပယ်ခြင်း ခံရမည်။ (eg. Fart , Burp , Flashlight ,Kama Sutra Apps)

  • 2.12 သင်၏ App သည် User များအတွက် အသုံးမဝင်ခြင်း (သို့) Website ကို UIWebView ဖြင့်သာ သာမန် Load လုပ်ထားသော App များ ၊ ကြာရှည် အသုံးပြုလို့ မကောင်းသော App ဖြစ်ပါက ငြင်းပယ်ခြင်း ခံရမည်။

  • 2.13 သင်၏ App သည် ကြော်ငြာသီးသန့် (သို့) စျေးကွက်သီးသန့် ဖြစ်ပါက ငြင်းပယ်ခြင်း ခံရမည်။

  • 2.14 သင်၏ App သည် သင်မဖော်ပြသော Trick (သို့) Fake Function များပါက ငြင်းပယ်ခြင်း ခံရမည်။

  • 2.15 သင်၏ App သည် Celluar Network ကိုအသုံးပြုပြီး Internet ပေါ်မှ 20MB Download ပြုလုပ်ထားပါက ငြင်းပယ်ခြင်းခံရမည်။

  • 2.16 သင်၏ App သည် Multitasking App ဖြစ်ပါက Background Service အနေနဲ့ သတ်မှတ်ထားသော VoIP , Audio Playback ,Location , Task Completion , Local Notification အပိုင်းများကို အသုံးပြုရမည်ဖြစ်သည်။

  • 2.17 သင်၏ App သည် Web Browsing Feature ပါဝင်ပါက iOS WebKit Framework နှင့် WebKit Javascript ကိုသာအသုံးပြုရမည်ဖြစ်သည်။

  • 2.18 သင်၏ App သည် Alcohol (သို့) တားမြစ်မူးယစ်ဆေးဝါး များ (သို့) ဆေးလိပ်သောက်ခြင်းများကို အလွန်အကျွံ အားပေးအားမြှောက်ပြုလုပ်ခြင်းများ ပါရှိပါက ငြင်းပယ်ခြင်း ခံရမည်။

  • 2.19 သင်၏ App သည် မမှန်ကန်သော အချက်အလက် (သို့) မတိကျသော Device Data များပါဝင်ပါက ငြင်းပယ်ခြင်း ခံရမည်။

  • 2.20 သင်သည် App Store ပေါ်သို့ တူညီသော App (Similar Apps) များဖြင့် Spamming ပြုလုပ်ခဲ့ပါက iOS Developer Program မှထုတ်ပယ်ခြင်း ခံရမည်ဖြစ်သည်။

  • 2.21 သင်၏ App သည် သီချင်းတစ်ပုဒ် (သို့) ရုပ်ရှင်တစ်ခု ဖြစ်ပါက iTunes Store ပေါ်သို့သာ Submit ပြုလုပ်ရမည်။ သင်၏ App သည် စာအုပ် တစ်အုပ် ဖြစ်ပါက iBook Store ကိုသာ Submit ပြုလုပ်ရမည်ဖြစ်သည်။

  • 2.22 သင်၏ App သည် သင်စိတ်ကူးပေါက်သလို User များကို အသုံးပြုခွင့် ကန့်သတ်မှုများ (Location , Carrier) ပါရှိပါက ငြင်းပယ်ခြင်း ခံရမည်။

  • 2.23 သင်၏ App သည် Data Store Feature ပါရှိပါက iOS Data Storage Guidelines အတိုင်းလိုက်နာရမည်။ မလိုက်နာပါက ငြင်းပယ်ခြင်း ခံရမည်။

  • 2.24 သင်၏ App သည် NewsStand အတွက်ဖြစ်ပါက Developer Program License Agreement ၏ Schedule 1 ,2 ,3 ကိုလိုက်နာရမည်။ မလိုက်နာပါက ငြင်းပယ်ခြင်းခံရမည်။

3. Metadata (name , descriptions , ratings , rankings , etc)

  • 3.1 သင်၏ App ၏ Metadata သည် အခြားသော Mobile Platform နာမည်များ အကြောင်းပြောဆိုခြင်း (သို့) ဖော်ပြခြင်း ပါရှိပါက ငြင်းပယ်ခြင်းခံရမည်။

  • 3.2 သင်၏ App သည် Placeholder text ပါရှိပါက ငြင်းပယ်ခြင်းခံရမည်။

  • 3.3 သင်၏ App ၏ Descriptions သည် App တွင်ပါရှိသော လုပ်ဆောင်ချက်များနှင့် ဆက်စပ်မှုမရှိပါက ငြင်းပယ်ခြင်း ခံရမည်။

  • 3.4 သင်၏ iTunes Connect ပေါ်တွင် ပြထားသော App Name နှင့် Device ပေါ်တွင်ပြထားသော App Name ရှုပ်ထွေးမှုမရှိစေရန် တူညီအောင် ဆောင်ရွက်ထားရမည်။

  • 3.5 သင်၏ App ၏ Small iCon ပုံစံ နှင့် Large iCon ပုံစံကို တူညီအောင်ရေးဆွဲထားရမည်။

  • 3.6 သင်၏ App နှင့် သက်ဆိုင်သော App iCon နှင့် Screenshot များသည် Rating အနေဖြင့် 4+ မရှိပါက ငြင်းပယ်ခြင်း ခံရမည်။

  • 3.7 သင်၏ App သည် သင်လုပ်ဆောင်ထားသော Feature များနှင့် မသက်ဆိုင်သော Category များရွေးချယ်ထားပါက ငြင်းပယ်ခြင်းခံရမည်။

  • 3.8 သင်၏ App နှင့် သက်ဆိုင်သော Rating များအတွက် သင်သာလျှင် တာဝန်ရှိသည်ဖြစ်သည်။ထို့အပြင် Apple မှလိုအပ်ပါက ပြင်ဆင်ခွင့် ၊ ဖျက်သိမ်းခွင့်ရှိသည်။

  • 3.9 သင်၏ App နှင့်သက်ဆိုင်သော Keywords များအတွက် သင်သာလျှင် တာဝန်ရှိသည်ဖြစ်သည်။ထို့အပြင် Apple မှလိုအပ်ပါက ပြင်ဆင်ခွင့် ၊ ဖျက်သိမ်းခွင့်ရှိသည်။

  • 3.10 အကယ်၍သင်သည် Fake Review (သို့) Paid Review စသော တစ်စုံတစ်ရာ ဖြင့် Apple ၏ User Review (သို့) Chart Ranking ကို Cheat လုပ်ဖို့ကြိုးစားပါက iOS Developer Program မှထုတ်ပယ်ပစ်ခြင်း ခံရမည်ဖြစ်သည်။

  • 3.11 သင်၏ App ကို User မှ Install ပြုလုပ်သောအခါ (သို့) Launch ပြုလုပ်သောအခါတွင် iOS Devices ကို Restart ချခိုင်းပါက ငြင်းပယ်ခြင်း ခံရမည်ဖြစ်သည်။

  • 3.12 သင်၏ App ကို Review တင်သွင်းသည့်အခါမှာ ပြည့်စုံသော ဆက်သွယ်ရန် လိပ်စာများ ၊ Support URL (or) Website များ ၊ Privacy policy URLs များကို အတတ်နိုင်ဆုံး ပြည့်စုံအောင် ဖြည့်သွင်းရပါမည်။

4. Location

  • 4.1 သင်၏ App သည် User ထံမှ User ၏ တည်နေရာ အသုံးပြုခွင့် မတောင်းခံပဲနှင့် အသုံးပြုခဲ့ပါက ငြင်းပယ်ခြင်းခံရမည်။

  • 4.2 သင်၏ App သည် လေယျာဉ် ၊ မော်တော်ကား နှင့် အခြားသော Devices တွေကို အလိုအလျောက် ထိန်းချုပ်စနစ် ပါရှိသော Location-based API များကိုအသုံးပြုထားပါက ငြင်းပယ်ခြင်းခံရမည်။

  • 4.3 သင်၏ App သည် Emergency Services (သို့) Fleet Management များကို အရေးပေါ်အခြေအနေ ပို့လွှတ်သော စနစ်များပါရှိသော Location-based API များကိုအသုံးပြုထားပါက ငြင်းပယ်ခြင်းခံရမည်။

  • 4.4 သင်သည် Location Data Services ကိုအသုံးပြုမည်ဆိုပါက User ၏ခွင့်ပြုချက်များကို တောင်းခံပြီး သင်၏ App နှင့်သာ သက်ဆိုင်သော Features များနှင့် သင်၏ ကြော်ငြာများအတွက်သာ အသုံးပြုခွင့်ရှိပါသည်။

ဒီတစ်လတော့ အထက်ပါ အကြောင်းအရာများကို ရေးသားပေးလိုက်ပါတယ်။နောက်လများမှာလည်း အပိုင်းဆက် ဖြစ်သော “App Store ပေါ်မှ စည်းမျဉ်းများ အပိုင်း (၂)” ကိုဆက်လက် စောင့်မျှော် ရှုစားပေးပါဦးလို့ တိုက်တွန်းရင်းနဲ့ပဲ နှုတ်ခွန်းဆက်သပါတယ်။ အချိန်ပေး ဖတ်ရှုပေးခြင်းကို ကျေးဇူးအထူးတင်ပါတယ်။
