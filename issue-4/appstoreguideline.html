App Store Review Guidelines (Part 2)

App Store ပေါ်မတင်ခင် သိသင့်သည့် စည်းမျဉ်းများ
ရေးသားသူ : ဇော်မင်းစိုး

ဒီတစ်လမှာတော့ အရင်လက ကျွန်တော်ရေးခဲ့တဲ့ App Store ပေါ်မှာ စည်းမျဉ်းများ ကိုအပိုင်း(၂) ဖြင့်ဆက်လက်တင်ဆက်လိုက်ပါတယ်။

5. Push Notifications

5.1 သင်၏ App သည် Apple ၏ Push Notifications APN (API) ကိုအသုံးမပြုပဲ အခြားသော API ဖြင့် Push Notification ပြုလုပ်ထားပါက ငြင်းပယ်ခြင်းခံရမည်။

5.2 သင်၏ App သည် Apple မှ Push Applications ID မပါပဲနှင့် Push Notification ပြုလုပ်ထားပါက ငြင်းပယ်ခြင်းခံရမည်။

5.3 သင်၏ App သည် Push Notification ပြုလုပ်လိုပါက ပထမဦးဆုံးအကြိမ် User ၏ခွင့်ပြုချက်ရယူရမည်။မရယူပဲ Push Notification ပြုလုပ်ပါက ငြင်းပယ်ခြင်း ခံရပါမည်။

5.4 သင်၏ App သည် Personal အကြောင်းအရာများ (သို့) လျှို့ဝှက် သတင်းအချက်အလက်များကို Push Notification ပြုလုပ်ပါက ငြင်းပယ်ခြင်းခံရပါမည်။

5.5 သင်၏ App သည် User ထံသို့ မဆီလျှော်သော အကြောင်းအရာများ ၊ Spamming ပို့ခြင်းများ စသည်တို့ပါရှိပါက ငြင်းပယ်ခြင်းခံရပါမည်။

5.6 သင်၏ App သည် Advertising , promotions နှင့်အခြားအခြားသော Marketing နှင့်သက်ဆိုင်သော အရာများကို Push Notification အသုံးမပြုရပါ။

5.7 သင်၏ App သည် User အား Push Notification သုံးကိုသုံးရမည်ဟု တိုက်တွန်းခြင်းမပြုရ။

5.8 သင်၏ App သည် APN မှသတ်မှတ်ထားသော Network Bandwidth ထက်ပိုမိုသုံးစွဲထားခြင်း ၊ Device တွေကို အလွန်အကဲ Burden ဖြစ်စေသော လုပ်ဆောင်ချက်များပါရှိပါက ငြင်းပယ်ခြင်းခံရမည်။

5.9 သင်၏ App သည် Virus များပါဝင်သော Code များ (သို့) ဖိုင်များ ၊ Program များ နှင့် Apple ၏ APN Services ကိုဖြတ်တောက်ဖြိုခွဲခြင်းများ ပါရှိပါက ငြင်းပယ်ခြင်းခံရပါမည်။

6. Game Center

6.1 သင်၏ App သည် မည်သည့် Player ID ကို End User ထံသို့ (သို့) Third Party သို့ ဖော်ပြခြင်းများပါရှိပါက ငြင်းပယ်ခြင်းခံရမည်။

6.2 သင်၏ App သည် Player ID ကို Game Center မှ ချမှတ်ထားသော Terms များမှအပ သွေဖယ်ပြီး အသုံးပြုလျှင် ငြင်းပယ်ခြင်းခံရမည်။

6.3 သင်သည် Game Center Player ID နှင့်ပက်သက်ပြီး look up , trace , relate ,associate , mine , harvest နှင့် အခြားသော မရိုးသားသော လုပ်ဆောင်ချက်များ လုပ်ဆောင်ထားပါက iOS Developer Program မှထုတ်ပယ်ခြင်းခံရမည်။

6.4 Game Center နှင့်သက်ဆိုင်သော အချက်အလက်များ (eg.Leaderboard score) ကို Apple မှ Game Center အတွက် ခွင့်ပြုမှသာ အသုံးပြုခွင့်ရှိပါသည်။

6.5 သင်၏ App သည် Game Center သို့မမှန်မကန်သော Message များပို့လွှတ်ခြင်း နှင့် အခြားသော မရိုးသားသော Game Center ထိခိုက်စေသော အကြောင်းအရာများ ပြုလုပ်ခြင်းပါဝင်ခဲ့ပါက ငြင်းပယ်ခြင်းခံရမည်။

6.6 သင်၏ App သည် Game Center မှသတ်မှတ်ထားသော Network Bandwidth ထက်ပိုမိုသုံးစွဲထားခြင်း ၊ Device တွေကို အလွန်အကဲ Burden ဖြစ်စေသော လုပ်ဆောင်ချက်များပါရှိပါက ငြင်းပယ်ခြင်းခံရမည်။

6.7 သင်၏ App သည် Virus များပါဝင်သော Code များ (သို့) ဖိုင်များ ၊ Program များ နှင့် Apple ၏ Game Center ကိုဖြတ်တောက်ဖြိုခွဲခြင်းများ ပါရှိပါက ငြင်းပယ်ခြင်းခံရပါမည်။

7. iAd

7.1 သင်၏ App ကို လူအထင်ကြီးစေရန်အတွက် ဟန်ဆောင်ထားသော Ads များကဲ့သို့ Ads များပါဝင်ခဲ့ပါက ငြင်းပယ်ခြင်းခံရမည်.။

7.2 သင်၏ App သည် Empty iAd Banner ပါဝင်ခဲ့ပါက ငြင်းပယ်ခြင်းခံရမည်။

7.3 သင်၏ App သည် iAd ပေါင်းမြောက်များစွာပါဝင်ခဲ့ပါက ငြင်းပယ်ခြင်းခံရမည်။

8. Trademarks and Trade dress

8.1 သင်၏ App သည် Apple မှချမှတ်ထားသော Term and conditions နှင့် Guideline များကို လုံးဝလိုက်နာရမည်။

8.2 သင်၏ App တွင်ပါရှိသော လုပ်ဆောင်ချက်များနှင့် အခြားသော အချင်းအရာများကို Apple မှ ထောက်ခံပေးထားခြင်း (သို့) Apple မှ Source Supplier အဖြစ်ပြုလုပ်ပေးထားသော အချက်အလက်များပါဝင်နေပါက ငြင်းပယ်ခြင်းခံရမည်။

8.3 သင်၏ App သည် Apple မှထုတ်လုပ်ထားသော App များဖြင့် တူညီနေခြင်း (သို့) Apple ၏ ကြော်ညာ Theme များနှင့် တူညီနေပါက ငြင်းပယ်ခြင်းခံရမည်။

8.4 သင်၏ App သည် Apple ၏ Product name များကို misspell ပြုလုပ်ထားခြင်း (eg. GPS For iPhone , iTunz) များဖြစ်နေပါက ငြင်းပယ်ခြင်းခံရမည်။

8.5 Third Party Material များဖြစ်သော (Trademarks , copyrights , trade secrets) နှင့် အခြားသောအချက်အလက်များကို Request ပြုလုပ်သည့်အပေါ်မှာ မူတည်ပြီး လက်ခံစဉ်းစားပေးသွားမည်။

8.6 Google Map API မှရယူထားသော Google Maps နှင့် Google Earth ပုံရိပ်များကို အသုံးပြုခဲ့ပါက မူလ Google ၏ အချက်အလက်များကို မည်သည့်ပြုဆင်မှုမှမပါဝင်ပဲ မူလအတိုင်း ထင်သာမြင်သာအောင် ဆောင်ရွက်ပေးထားရမည်။ သင်၏ App သည် Google Logo သို့ Copyright အချက်အလက်များကို တစ်စုံတစ်ရာ ပြုပြင်ခြင်းများပါရှိခြင်း ၊ ဖုံးကွယ်ထားခြင်းများ ပါရှိပါက ငြင်းပယ်ခြင်းခံရမည်။

9. Media Content

9.1 သင်၏ App သည် Apple မှချမှတ်ပေးထားသော Media Framework ကိုအသုံးမပြုပဲ Music Library ကိုအခြားသောနည်းလမ်းဖြင့် Access လုပ်ထားပါက ငြင်းပယ်ခြင်းခံရမည်။

9.2 သင်၏ App သည် Any iPod User Interface ကိုဆင်တူရိုးမှား ပြုလုပ်ထားခြင်းမှာပါဝင်ခဲ့ပါက ငြင်းပယ်ခြင်းခံရမည်။

9.3 Audio Streaming ကိုအသုံးပြုထားပါက Cellular Data ပေါ်တွင် 5 minutes အတွင်း 5MB ထက်ကျော်လွန်ပြီး မလုပ်ဆောင်ရပါ။

9.4 Video Streaming ကိုအသုံးပြုထားပါက Cellular Data တွင် 10 minutes ထက်ပိုမိုပါက HTTP Live Streaming နှင့် Baseline 64 ops Audio Only HTTP Live Stream ကိုအသုံးပြုရမည်ဖြစ်ပါသည်။

10. User Interface

10.1 သင်၏ App သည် Apple မှချမှတ်ထားသော Apple iOS Human Interface Guidelines မှ Rules များကို လိုက်နာထားရမည်ဖြစ်သည်။

10.2 သင်၏ App သည် Apple ၏ iPhone တွင်ပါဝင်သော မူလ App များ ၊ App Store ပေါ်မှာ Apple ၏ App များနှင့်ဆင်တူရိုးမှား ပြုလုပ်ထားခြင်းများ ပါရှိပါက ငြင်းပယ်ခြင်းခံရမည်။

10.3 သင်၏ App သည် Apple iOS Human Interface Guidelines မှသတ်မှတ်ထားသော စည်းမျဉ်းများမှ သွေဖယ်သော Button ပုံစံများ ၊ iCon ပုံစံများ အသုံးပြုထားပါက ငြင်းပယ်ခြင်းခံရမည်။

10.4 သင်၏ App သည် Desktop Home screen ကဲ့သို့ Multi-App widget များကဲ့သို့ လုပ်ဆောင်ချက်များ ပါဝင်ခဲ့ပါက ငြင်းပယ်ခြင်းခံရမည်။

10.5 သင်၏ App သည် Standard switch Control ၏ မူလလုပ်ဆောင်ချက်များကို သွေဖယ်ပြီးလုပ်ဆောင်ထားပါက ငြင်းပယ်ခြင်းခံရမည်။

10.6 Apple နှင့် Apple User များသည် ရိုးရှင်းမှု ၊ သန့်စင်မှု ၊ တီထွင်ဆန်းသစ်မှုများကို နှစ်သက်သဘောကျပါသည်။ ထို့ကြောင့် သင်၏ User Interface သည် ရှုပ်ထွေးပွေလီ နေပါက ငြင်းပယ်ခြင်းခံရမည်။

ဒီတစ်လတော့ ဒီလောက်နဲ့ပဲ ရပ်နားရအောင်။ နောက်တစ်လမှာတော့ အပြီးသတ်ရေးပေးမဲ့အပြင် ကျွန်တော့်ရဲ့ App Store Review Guideline နဲ့ပက်သက်ပြီး အတွေ့အကြုံတွေကိုပါ ဆက်လက်ရေးသားပေးသွားမှာဖြစ်ပါတယ်။
