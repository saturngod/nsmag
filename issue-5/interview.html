Interview with Ye Myat Min

Nex Co.,Ltd Founder ပြောတဲ့ ရန်ကုန်က startup ဘဝ
ရေးသားသူ : saturngod

၁။ မင်္ဂလာပါ ရဲမြတ်မင်းရေ။ အရင်ဆုံး ကိုယ့်ကိုယ် မိတ်ဆက်ပေးပါအုံး။

ကျွန်တော့် နာမည် ရဲမြတ်မင်းပါ။ အသက်က ၂၂ ပေါ့။ Nex Co.,Ltd ရဲ့ founder ပါ။ Singapore Managment University မှာ degress course ကို တက်ရောက် သင်ကြားနေနေပါတယ်။ ငယ်ငယ်ကတည်းက computer တွေနဲ့ ရင်းနှီးကျွမ်းဝင်ခဲ့ပါတယ်။ အသက် ၁၄ နှစ် ကတည်းကစပြီးတော့ programming ကို စတင်စိတ်ဝင်စားခဲ့တယ်။ အသက် ၁၉ နှစ်မှာ စွန့်ဦး တီထွင်သူတစ်ယောက် , product design တွေ ဖန်တီးတဲ့ သူတစ်ယောက် အရမ်းဖြစ်ချင်ခဲ့တယ်။ အသက် ၂၁ နှစ်မှာ Nex Co.,Ltd ကို သူငယ်ချင်း တွေ နဲ့ အတူတူ စတင်တည်ထောင်ခဲ့ပါတယ်။ အရမ်းကို အဖိုးတန်တဲ့ အတွေ့အကြုံတွေ ရခဲ့ပါတယ်။

၂။ NexMM ကို buzzinmm ကနေ စပြီး သိခဲ့ပါတယ်။ NexMM စဖြစ်ခဲ့ပုံလေး ပြောပြပေးပါအုံး။

Buzzin က ကျွန်တော်တို့ စခဲ့တဲ့ အချိန်တုန်းက side project တစ်ခုပါ။ အဲဒီ project ကပဲ ကျွန်တော်တို့ ရှေ့ဆက် ဘာတွေလုပ်သင့်သလဲ ဆိုတာကို သတိပြုခဲ့တဲ့ project တစ်ခုပါ။ Buzzinmm မတိုင်ခင်က mobile နဲ့ web app တွေကို freelancers အနေနဲ့ ကော small team အနေနဲ့ပါ လုပ်ခဲ့ဖူးပါတယ်။ Buzzinmm ကို ဖန်တီးတဲ့ အခါမှာ product development , marketing နဲ့ customer relationshop စတာတွေကို လေ့လာဖြစ်ခဲ့ပါတယ်။ သို့ပေမယ့်လည်း အဲဒါက အောင်မြင်တဲ့ project တစ်ခု မဖြစ်ခဲ့ပါဘူး။ ကျွန်တော်တို့မှာ startup အနေနဲ့ အရေးကြီး လုပ်စရာလေးတွေ ရှိနေပါတယ်။ ကျွန်တော်တို့ angel investor ကို 2013 အစောပိုင်းမှာ တွေ့ခဲ့ပါတယ်။ မကြာခင်မှာ seed funding ရပြီးတော့ လူ ၉ ယောက်နဲ့ ၅ လ အတွင်းမှာ စခဲ့ပါတယ်။ mobile ကို အဓိကထားပြီးတော့ လှပသေသပ်တဲ့ product တွေကို ပြည်တွင်းမှာ ထုတ်ပြီးတော့ လူတွေ ရဲ့ နေ့စဉ် pixel တစ်ခု စာ အချိန်လေးတွေကို အကျိုးရှိအောင် လုပ်ပေးဖို့ အာရုံစိုက်ထားပါတယ်။

၃။ startup တစ်ခု ရန်ကုန်မှာ စတဲ့ အခါမှာ အဓိက စိန်ခေါ်မှုက ဘာလဲ။

roller coaster လိုပါပဲ။ စိန်ခေါ်မှုတွေကတော့ အများကြီးပေါ့ဗျာ။ ကျွန်တော်တို့ရဲ့ အဓိက စိန်ခေါ်မှုက ကျွန်တော် (founder) က ပြည်ပ မှ ရောက်နေတာပဲ။အခုအချိန်မှာ ကျွန်တော်က စလုံးမှာ နေထိုင်ပြီးတော့ အခြားသူတွေက ရန်ကုန်မှာ လုပ်နေကြတယ်။ အချင်းချင်း ဆက်သွယ်ရေးက အခက်အခဲလေးတွေ ရှိပါတယ်။ ကြီးကြီးများများကြီး သက်ရောက်မှုတော့ မရှိပါဘူး။

company ကို မှတ်ပုံတင်ဖို့ လုပ်တုန်းက စိတ်ခေါ်မှုတွေကို ကြုံခဲ့ရပါတယ်။ step-by-step guides တွေက online ပေါ်မှာ မရှိပါဘူး။ အကုန်အကျများပြီးတော့ ပျင်းဖို့လည်း ကောင်းလှပါတယ်။ ဒါကြောင့် အတွေ့အကြုံရှိပြီးသား agency က ကူညီပေးခဲ့ပါတယ်။ ကျွန်တော့် အမြင်ကတော့ company တစ်ခုစဖို့ကို လွယ်လွင့် တကူလုပ်ထားပေးသင့်တယ်။ ဒါမှသာ အခု မှ စလုပ်မယ့် သူတွေအတွက် စိတ်အားထက်သန်ပြီးတော့ စွန့်ဦးတီထွင်သူတွေ ရှိလာအောင် ဆွဲဆောင်နိုင်ပါလိမ့်မယ်။

နည်းပညာနဲ့ ပတ်သက်တဲ့ သူတွေနဲ့ စကားပြောတိုင်း ကျွန်တော် Myanmar နိုင်ငံမှာ နည်းပညာ company တစ်ခု စ တင်နေတယ်ဆိုတာကို အံဩကြတယ်။ အမြဲတန်းမေးနေကျက အင်တာနက် connection ကိစ္စတွေ ဘယ်လို လုပ်မလဲ ဆိုတာပဲ။ ဟုတ်တော့ဟုတ်ပါတယ်။ အဲဒီ ပြဿနာက ရန်ကုန် မှာ ရှိတဲ့ startup တွေအတွက် အဟန့် အတား တစ်ခုပါပဲ။ ဒါပေမယ့် ဒါဟာ အနှောင့်အယှက် တစ်ခုပါ။ တကယ်လာပြီး လုပ်ကြည့်သင့်ပါတယ်။ လုပ်နေရင်းနဲ့ ဒါဟာ used to ဖြစ်သွားပါတယ်။ ကျွန်တော်တို့မှာလည်း ပြဿနာ တွေရှိပါတယ်။ Private Message tools တွေက သုံးနေရင်းနဲ့ ရပ်ရပ်သွားပါတယ်။ တစ်ခါတစ်လေ Github/Bitbucket ပေါ်ကို အချိန်မှီ တင်မရလို့ deadline တွေနဲ့ လွှဲသွားတာတွေ ရှိပါတယ်။ ဒါတွေဟာ ခဏတာ လို့ ယုံကြည်ပါတယ်။ ဒါတွေကို ကျွန်တော်တို့ကို risk management ပိုကောင်းအောင် ကူညီပေးတယ်လို့ ထင်ပါတယ်။

တခြား ပြဿနာ တစ်ခုကတော့ ရုံးခန်း ငှားရတာပဲ။ ငှားခ စျေးနှုန်း တွေက မိုးပေါ်ကို တက်နေတယ်လို့ ထင်တယ်။ အိမ်ရှင်တွေကလည်း နိုင်ငံခြား compnay တွေ လာပြီး ငှားမှာ ကို စောင့်နေပါတယ်။ local startup တွေနဲ့ ဒီလောက် စျေးနှုန်းတွေကို မပေးနိုင်ပါဘူး။ အခန်းငှားခ စျေးနှုန်းတွေက startup အနေနဲ့ ရင်ဆိုင်ရတဲ့ စိန်ခေါ်မှု ပါပဲ။ အထူးသဖြင့် အခု အချိန်မှ စပြီး လုပ်မယ့် startup တွေပေါ့။ ကျွန်တော် ယုံကြည်တာကတော့ လူငယ်တွေ အနေနဲ့ စပြီး လုပ်ဖို့ရာမှာ ကြီးမားတဲ့ အခက်အခဲ တစ်ခုပါပဲ။ ကံကောင်းတာကတော့ Devspace က ကျွန်တော်တို့ကို ကူညီ ခဲ့ပါတယ်။ သူတို့ကူညီခဲ့တဲ့ ကျေးဇူးတင်လို့ မဆုံးနိုင်ပါဘူးဗျာ။

နောက်ပြီး ရန်ကုန်မှာ အလုပ်အတွက် ပါရမီ ရှင်တွေ ရဖို့ အတော် ခက်ပါတယ်။ နိုင်ငံခြား company တွေက အခုအခါမှာ experience ရှိပြီးသားလူတွေ အရည်အချင်းရှိတဲ့ သူတွေကို လခများများပေးပြီး ခန့်လာကြပါတယ်။ HR ပိုင်းက မြန်မာနိုင်ငံမှာ တကယ့်ကို ရင်ဆိုင်ရတဲ့ စိန်ခေါ်မှု တစ်ခုပါပဲ။ ဒါပေမယ့် ကျွန်တော့်မှာတော့ ကျွန်တော် နဲ့ အတူတူ လုပ်မယ့် အဖွဲ့ကောင်းကောင်း ရှိပါပြီ။

၄။ လက်ရှိ Project Management အတွက် ဘယ် methology ကို အသုံးပြုလဲ။ ( Agile , SCRUM , etc) ။ ဘာကြောင့် အသုံးပြုတာလဲ။

လက်ရှိတော့ agile project managment ကို သုံးနေတယ်။ ကျွန်တော်တို့ အချိန် တိုတိုနဲ့ အကျိုးရှိနိုင်မယ့် managment ကို အမြဲ ရှာဖွေနေပါတယ်။ အခု agile ကို သုံးနေပါတယ်။ ဘာလို့လည်း ဆိုတော့ agile ဟာ continuously deliver လုပ်နိုင်ပြီးတော့ လက်ရှိ project ဘာတွေ ပြောင်းလဲရမယ် ဆိုတာကို သိနိုင်တယ်။ ကျွန်တော် စင်္ကာပူမှာ ရှိနေပြီးတော့ ကျွန်တော်တို့တွေဟာ online tools တွေ အကူအညီနဲ့ ဆက်သွယ်ကြသလို စီမံခန့်ခွဲ မှုတွေလည်း လုပ်ကြပါတယ်။ ကျွန်တော် တို့ ကြိုက်ပြီး လက်ရှိ သုံးနေတဲ့ tools တွေကတော့

စတာတွေ ကို သုံးပါတယ်။

၅။ Company မှာ အဓိက Ruby On Rails အသုံးပြုတယ်လို့ သိရပါတယ်။ ဘာဖြစ်လို့ ROR ကို ရွေးချယ်ရတာလဲ။

ဘာလို့လည်းဆိုတော့ ကျွန်တော်က Ruby fan ဖြစ်နေလို့ပါပဲ။ နောက်တာပါဗျာ။ အဓိကတော့ ROR ဟာ mature ဖြစ်ပြီး app ကို တည်ဆောက်ရာမှာ မြန်ဆန်စေလို့ပါပဲ။ ဒါ့အပြင် ROR community ဟာ မယုံနိုင်စရာပါပဲ။ သူတို့ရဲ့ အကူအညီတွေ ကျွန်တော်တို့တွေ အများကြီး ရပါတယ်။ ROR ဟာ API တွေကို လွယ်လင့် တကူ တည်ဆောက်နိုင်တဲ့ အတွက်ကြောင့် mobile products တွေ client projects တွေ အတွက် အသုံးဝင်လှပါတယ်။

ROR အပြင် ကျွန်တော်တို့တွေဟာ javascript-base web tech တွေကိုလည်း သဘောကျပါတယ်။ Node.js , Meteor js တို့ပေါ့ဗျာ။

၆။ NexMM ရဲ့ Mobile app CMS product အကြောင်းကို လူ စိတ်ဝင်စားကြပါတယ်။ CMS system ဖြစ်တဲ့ အတွက်ကြောင့် server side မှာ host လုပ်မယ်လို့ ထင်ပါတယ်။ အဲဒီအတွက် server side scaling ကိစ္စ တွေကို ဘယ်လို စီစဉ်ထားလဲ။

ဟုတ်ပါတယ်။ ကျွန်တော်တို့တွေဟာ စီးပွားရေးအတွက် အလွယ်ဆုံး နဲ့ လှပသေသပ်ပြီး native , offline mobile apps တစ်ခုကို အသက်အသာဆုံး ဖန်တီးပေးမယ့် cms product တစ်ခုပါ။ အဲဒီ မှာ အပိုင်း ၂ ပိုင်းပါပါတယ်။ Web-based CMS နဲ့ mobile app အပိုင်းပါ။ အစ်ကိုပြောသလိုပါပဲ ကျွန်တော်တို့ CMS ကို Heroku ပေါ်မှာ တင်ဖို့ အစီအစဉ် ရှိပါတယ်။ အခုတော့ testing နဲ့ staging အပိုင်းမှာ ကျွန်တော်တို့တွေ digital ocean server တွေကို အသုံးပြုထားပါတယ်။ ကျွန်တော်တို့ Heroku ကို အသုံးပြုမယ်ဆိုရင် scaling အပိုင်းတွေ စဉ်းစားစရာ မလိုပဲ ကျွန်တော်တို့ရဲ့ MVP ကို အာရုံစိုက်နိုင်ပါလိမ့်မယ်။ သေချာတာကတော့ ကျွန်တော်တို့တွေ ဒီထက် ပိုပြီး ကြီးလာရင်တော့ Heroku ပေါ်မှာ တင်ဖို့ လုပ်ရပါလိမ့်မယ်။ အခုတော့ ကျွန်တော်တို့တွေဟာ product ကို ထုတ်ဖို့ အရင် ဦးစားပေးထားပါတယ်။
